# Questions Asked in the Interview

## 1. Basic Introduction

1. Briefly introduce yourself.
    
2. Tell me about your **personal background**.
    
3. Did you build your projects **individually or as part of a team**?
    
4. Are you comfortable working in **both frontend and backend**?
    

---

# 2. AI Usage in Development

5. Did you use **AI tools to build the assignment**?
    
6. Do you **fully understand the logic** of the code generated by AI?
    
7. Approximately **what percentage of the assignment was AI-generated**?
    

---

# 3. Spring Security / Backend

8. How do you **implement Role-Based Access Control (RBAC)** in Spring Boot?
    
9. What are the **steps involved in implementing role-based security** using Spring Security?
    
10. After implementing the **SecurityFilterChain**, how do you apply **role restrictions on APIs**?
    
11. How do you enable **method-level security** in Spring Boot?
    
12. How would you apply **role restrictions to a controller endpoint**?
    
13. If an API should be accessible to **multiple roles (ADMIN and VIEWER)**, how would you implement that?
    

---

# 4. Internship Discussion

14. What exactly did you **learn during your internship**?
    
15. What **assignments or projects** were given to you during the internship?
    
16. Can you explain a **complex API you implemented** during the internship?
    

---

# 5. Redis / Performance

17. Why did you use **Redis caching** in your project?
    
18. If the system can handle requests without Redis, **why add Redis at all**?
    
19. Did your team perform **benchmark testing** before introducing Redis?
    
20. At what **scale of traffic** would Redis actually be necessary?
    

---

# 6. Scheduling / Background Jobs

21. Do you know how to implement **Schedulers in Spring Boot**?
    
22. Can you **write code for a scheduled task**?
    

---

# 7. System Design / Live Coding

The interviewer asked you to **design a mini order management system**.

23. Design APIs for a system that supports:
    
- Creating products
    
- Creating orders
    
- Adding products to orders
    

---

# 8. Database Design

24. How would you design the **database schema for orders and products**?
    
25. Is it a good idea to **store a list inside a relational database column**?
    
26. How would you design a **many-to-many relationship between orders and products**?
    
27. Why should we store **created and modified timestamps** in database tables?
    

---

# 9. API Design Best Practices

28. Why are you using the **entity directly as API input** instead of using DTOs?
    
29. How would you implement **filters and pagination** in an API?
    

Example requirement given:

- Filter orders by **date range**
    
- Filter by **product name**
    
- Return **paginated results**
    

---

# 10. JPA / SQL

30. How would you write a **JPA query for filtering orders by product name**?
    
31. How would you **join order, order-product, and product tables** to fetch data?
    

---

# 11. Logical Puzzle

32. Bridge Crossing Puzzle:
    

Four people must cross a bridge at night.

Time required:

- 1 minute
    
- 2 minutes
    
- 7 minutes
    
- 10 minutes
    

Rules:

- Only 2 people can cross at once
    
- They must carry a flashlight
    
- When two cross together, they move at the **slower person's speed**
    

Question:

> How can all four cross the bridge in **17 minutes**?

---

# Real Observation

This interview tested **4 major areas**:

1️⃣ **Spring Security implementation**  
2️⃣ **Database design thinking**  
3️⃣ **Backend API design**  
4️⃣ **Problem-solving ability**

This is actually **closer to a 1–2 year developer interview**, not typical fresher level.
