## Month 1–3

- Understand full codebase.
    
- Ask seniors how billing flow works end-to-end.
    
- Learn how requests move from Controller → Service → Repository → DB.
    
- Master debugging logs.
    

Goal: Stop being “fresher”. Become “reliable junior”.

---

## Month 4–6

- Take ownership of small features.
    
- Handle at least 1 production issue alone.
    
- Learn DB indexing & performance basics.
    
- Understand transactions and concurrency.
    

Goal: Be able to explain system in interviews.

---

## Month 7–9

- Start preparing for next switch quietly.
    
- Revise:
    
    - Core Java (collections, multithreading basics)
        
    - Spring Boot internals
        
    - JPA/Hibernate
        
    - REST best practices
        
- Build 1 strong side project.
    

---

## Month 10–12

- Start applying.
    
- Target 6–8 LPA roles.
    
- Use real experience examples in interviews:
    
    - “In telecom billing, we handled high volume…”
        
    - “We optimized this query…”
        
    - “We fixed this production bug…”
        

That’s how salary jumps happen.
